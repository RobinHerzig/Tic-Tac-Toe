A very simple children's game built to understand Node.js without the convenience of Express.

## How It's Made:

Although this project was both front-end and backend, the focus was primarily on the latter. More specifically, we sought to learn more about how Node.js functions WITHOUT using Express or any modules.

To challenge ourselves, we required that at least some of the game logic must be performed server-side. We fulfilled this by writing an algorithm that takes the player's input, calculates the computer's choice, and determines the winner. This information is included in the response back to the client.

## Optimizations

Our goal was to create a functional game within the limitations we set for ourselves, and we achieved that. Next, more consideration would be put into expanding the game into a more original product. Perhaps, for example, we may include additional games and keep track of wins throughout all of them.

## Lessons Learned:

This was a group project, created with a few friends of mine. It was a great lesson in working as a team. Everyone had a lot to contribute and it was important to keep an open mind and remain flexible to other ideas.

Creating gave everyone on the team a new appreciation for Express. The methods and shortcuts it provides make it truly valuable in any backend workflow.
